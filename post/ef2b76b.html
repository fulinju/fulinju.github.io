四、协作开发建议

  1. 分支策略：主分支（main/master）仅用于发布稳定版本，新功能在 dev 或 feature 分支开发。
  2. 提交规范：commit 信息需清晰描述修改内容，推荐使用 Conventional Commits 格式。
  3. 冲突预防：频繁使用 git pull --rebase 保持本地与远程同步，减少合并冲突。
