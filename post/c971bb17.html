跨平台 GitHub SSH 配置指南（macOS & Windows）

一、准备工作

  1. 安装 Git 客户端
    • macOS：通过 Homebrew 或 Git 官网 下载安装包
    • Windows：从 Git 官网 下载安装程序，安装时勾选“Add Git to PATH”
  2. 注册 GitHub 账号
    • 访问 GitHub 官网 完成注册。

二、SSH 密钥生成与配置

1. macOS 配置步骤

(1) 生成 SSH 密钥

  • 按回车使用默认路径 ~/.ssh/id_ed25519 或自定义文件名
  • 设置密钥密码（可选，建议增强安全性）

  • 设置密码（可选）

(2) 添加密钥到代理

(3) 复制公钥

三、添加公钥到 GitHub

  1. 登录 GitHub → 点击右上角头像 → Settings → SSH and GPG Keys
  2. 点击 New SSH Key → 输入标题（如“My MacBook”）
  3. 粘贴复制的公钥内容 → Add SSH Key

四、测试 SSH 连接

  • 首次连接输入 yes
  • 成功提示：Hi username! You've successfully authenticated...

六、多账户管理（可选）

创建 ~/.ssh/config 文件（macOS/Windows 通用）：

七、常见问题

  1. Permission denied (publickey)

    • 检查公钥是否完整粘贴到 GitHub
    • 重启 ssh-agent：eval "$(ssh-agent -s)" && ssh-add ~/.ssh/id_ed25519

  2. 旧系统兼容性问题

    • 使用 RSA 算法替代 ed25519

八、跨平台工具推荐

  • Tabby：支持 Windows/macOS 的 SSH 客户端，可管理多连接
  • VS Code：内置终端与 Git 可视化工具

通过以上步骤，您已实现 GitHub SSH 的跨平台安全连接。如需图文详细演示，可参考 GitHub 官方文档。
